Describe your tests visually

Your tests shouldn't need to know how your app is constructed. They should describe the user experience, leaving developers free to build and refactor as they see fit.

The default client is a Python library, but you can use any language of your choice using a simple wrapper.


from chetbot import __

__('Username').tap()
keyboard.type_text('alice')

__('Password').tap()
keyboard.type_text('ilovebob')

keyboard.dismiss()

__(class='CheckBox').closest_to('Notifications').tap()

__('Sign in').tap()

Contiuous Integration made simple

Run tests in the cloud when you commit code, effortlessly. ChetBot gives you a one-line integration into your existing CI system and notifies you of any failures immediately.

Test reports that matter

Test reports tell you how to reproduce the problem, give you the logs that matter and video of the test that failed: All the information you need for bug tracking.
